## Interaction Flow

### 1. **First Visit (Wallet Disconnected)**
1. User sees welcome message and "Connect Wallet" button
2. User can explore all actions (but cannot execute)
3. User can view recent bets from other players
4. User can adjust bet parameters to see calculations

### 2. **Connecting Wallet**
1. User clicks "Connect Your Wallet to Play"
2. Modal appears with wallet options (Reown, MetaMask)
3. User selects wallet and authorizes connection
4. UI updates to show wallet address and balances

### 3. **Placing a Bet (Connected)**
1. User enters bet amount (or uses quick buttons)
2. User adjusts win chance slider or enters threshold
3. UI shows real-time calculation of multiplier and payout
4. User clicks "Place Bet"
5. Wallet prompts for transaction approval
6. Loading state shows transaction progress
7. Success/failure message appears
8. Recent bets table updates with new bet

### 4. **Managing Liquidity**
1. User navigates to Add/Remove Liquidity section
2. For removal, "MAX" button shows available liquidity
3. User enters amount and confirms
4. Transaction flow similar to betting

### 5. **Withdrawing Balance**
1. User sees available contract balance
2. User clicks "MAX" to withdraw all or enters custom amount
3. Confirms withdrawal
4. Funds transferred to wallet

## Technical Notes

### Formulas
```javascript
// Win chance to threshold
threshold = Math.floor(winChance * 65536 / 100)

// Threshold to win chance
winChance = (threshold / 65536) * 100

// Multiplier calculation
multiplier = (65536 / threshold) * (1 - houseEdge)

// Payout calculation
payout = betAmount * multiplier
```

### House Edge
- Default: 2% (0.02)
- Configurable per contract

### Balance Calculation
```javascript
availableBalance = walletBalance + contractBalance
```
